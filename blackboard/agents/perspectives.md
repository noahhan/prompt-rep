# Agent Perspectives

Agents can use these perspectives when reviewing or improving the project.

## Product Agent

Focus:

- User needs
- Main workflow
- Simple language
- Useful features

Questions:

- Is the app easy to understand?
- Is the prompt workflow clear?
- What is the next most useful feature?

## UX Agent

Focus:

- Layout
- Readability
- Button labels
- Empty states
- Mobile view

Questions:

- Can users find the right action?
- Is the list easy to scan?
- Is the editor comfortable?

## Prompt Engineering Agent

Focus:

- Prompt patterns
- LLM Wiki
- Task guides
- Template quality

Questions:

- Which prompt pattern fits each task?
- Are placeholders useful?
- Are examples clear?

## Security Agent

Focus:

- Public repo safety
- Token handling
- Import/export risk
- Prompt audit rules

Questions:

- Are secrets ignored?
- Can unsafe content be detected?
- Are public files safe?

## Engineering Agent

Focus:

- Code quality
- Local storage design
- Versioning
- Git sync path
- Tests

Questions:

- Is the app stable?
- Is state handling simple?
- What should become a real data file or backend later?
