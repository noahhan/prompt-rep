# Wiki Instructions for LLMs

Use these rules when updating this wiki.

## Workflow

1. Put unprocessed material in `00-inbox/`.
2. Summarize sources in `10-sources/`.
3. Extract ideas into `20-notes/`.
4. Move stable ideas into `30-concepts/`.
5. Turn useful ideas into task guides in `40-playbooks/`.
6. Record changes in `90-meta/changelog.md`.

## Writing Style

- Use simple English.
- Use short sections.
- Prefer examples over abstract explanation.
- Keep one idea per bullet.
- Mark uncertain claims as `Uncertain`.

## Source Rules

Each source note should include:

- Title
- URL
- Date added
- Main ideas
- Useful patterns
- Limits or cautions

## Concept Rules

Each concept should include:

- What it is
- When to use it
- When not to use it
- Template
- Example
- Related concepts

## Prompt Vault Rules

Knowledge should help the app answer:

- What kind of task is the user doing?
- Which prompt patterns fit this task?
- What placeholders are needed?
- What output format is best?
- What safety checks are needed?
